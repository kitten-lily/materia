# Buildbarn server TLS cert

`server.crt` is generated by `mise buildbarn:server-cert-init --hostname <hostname>`
and committed here — it's public (identity verification only, no secret
material). The matching private key (`server.key`) is never written to
disk in this repo; it goes straight into `attributes/<server>.yml` as
`components.buildbarn.serverKeyPem` (SOPS-encrypted) and is mounted into
the containers as a materia Secret at deploy time.

Unlike krytis's own local test design, there is no CA and no client
certs here — auth is JWT bearer token (see `.mise/tasks/buildbarn/
{secret-init,mint-token}`), not mTLS. This cert exists purely for
transport confidentiality through the tunnel.

Regenerating `server.crt` (e.g. because the DNS hostname changes) requires
removing `components.buildbarn.serverKeyPem` from the vault first —
`server-cert-init` is idempotent and skips generation otherwise.
